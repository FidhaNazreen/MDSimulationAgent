## Section 1 — Per-issue acknowledgments

**R2-1** (don't pass Traversable to sha256_dir; v0 only supports filesystem-backed installs).
**Accept.** `mdagent._resources.schemas_dir()` calls `importlib.resources.files("mdagent._resources.schemas")`, asserts the result is filesystem-backed (`hasattr(res, "_path")` or `isinstance(res, MultiplexedPath)` check), and returns a `pathlib.Path`. If it isn't filesystem-backed, raise `UnsupportedResourceInstall("zipapp/import-from-archive is unsupported; reinstall via `uv tool install`.")`. Document this constraint in the user-facing CLI errors and the doctor output.

**R2-2** (`as_file()` on directories is version-sensitive; not the core path).
**Accept.** We never call `as_file()`. The path returned by `files()` IS the on-disk path for normal wheel/editable installs; that's what we ship.

**R2-3** (canonical in-package resources; no build-time copy).
**Accept.** `src/mdagent/_resources/schemas/v0.1.0/*.json` is the source of truth. Top-level `schemas/` is **removed**, replaced by `schemas/README.md` (a one-line pointer file). No build hook, no symlinks. Tests load schemas via `mdagent.schemas.schemas_dir()`.

**R2-4** (no symlinks). **Accept.** Confirmed; the proposal doesn't use them.

**R2-5** (`install-skills` needs skills shipped in the wheel — repo `.claude/skills/` is invisible to the installed tool). **Accept — critical.** Skills move into `src/mdagent/_resources/skills/<name>/SKILL.md`. `install-skills` copies from `importlib.resources.files("mdagent._resources.skills")`. A repo-level `.claude/skills/` *also exists* (so Claude Code sees the project skills when working in this repo) and is generated by `mdagent install-skills --project .claude/skills/ --from-source` or by a tiny `make install-skills-repo` rule. Both paths produce identical content.

**R2-6** (don't ship `.claude/skills/` via packaging — dot-dirs may be excluded). **Accept.** Packaging is exclusively via `src/mdagent/_resources/skills/`. Repo's `.claude/skills/` is a build artifact, not a source. Smoke test: after wheel install, `mdagent install-skills --dry-run` lists every shipped skill.

**R2-7** (doctor check timing — network needed at ingest, not before topology). **Accept.** Restructuring:
  - `mdagent doctor` runs *config-aware* checks before orchestrator entry: derive what's needed from the resolved `run_config.json`, run only those checks.
  - Network check fires iff `input.pdb_id` is set AND `input.structure_path` is unset.
  - GROMACS check fires iff any step from `step_04_topology` onward will execute (always true in v0).
  - Viewer check fires iff `visualization.mode != "disabled"` AND `visualization.render in {"png", "both"}`.
  - The orchestrator calls `mdagent.doctor.check_for_run(cfg)` internally (not the CLI subcommand) so the check is enforced even when `run-workflow` is called from Python directly.

**R2-8** (viewer checks must not block script-only viz). **Accept.** Doctor's viewer entry is structured: `{viewer: vmd, binary_available: bool, renderer_available: bool, required_for_render_mode: bool}`. The "required" boolean reads `visualization.render`:
  - `state_only` → never required (scripts always written).
  - `png` or `both` → renderer required.
Doctor blocks the run only when `required_for_render_mode AND NOT renderer_available`.

**R2-9** (`doctor --json` contract inconsistent). **Accept.** Reversed: default is human-readable text on stdout; `--json` produces a single JSON object on stdout (nothing on stderr). No JSON-by-default. Suggestions live in `errors[i].suggestion` inside the JSON.

**R2-10** (`--suggest-install` conflicts with JSON). **Accept.** Dropping `--suggest-install` as a flag. Suggestions are *always* populated in the JSON's `errors[i].suggestion` field. In human mode, the suggestion is printed inline with the failure.

**R2-11** (PATH defense too optimistic; Claude Code may not have uv tool bin). **Accept.** README + skill body include:
  - Install snippet: `uv tool install git+https://...@v0.1.0 && uv tool dir --bin >>$SHELL_RC || …`
  - One-line check inside the skill: `command -v mdagent || echo "Install: see README; ensure $(uv tool dir --bin) is on PATH."`

**R2-12** (`uv tool upgrade` ambiguous when source is a pinned tag). **Accept.** README documents upgrades as:
  ```
  uv tool install --force git+https://github.com/<user>/MDSimulationAgent@<NEW_TAG>
  ```
  Not `uv tool upgrade mdagent` (which would re-install the same pinned tag).

**R2-13** (`prep-structure` should be a real subcommand). **Accept.** Implemented as a top-level `mdagent prep-structure ...` subcommand. Internally it reuses the `--stop-after prep` plumbing (so the same code path exists for users who want to stop mid-pipeline), but the documented + skill-referenced interface is `mdagent prep-structure`.

**R2-14** (`--no-doctor` is a footgun). **Accept.** Renaming to narrow flags:
  - `--skip-network-check`
  - `--skip-viewer-check`
  - `--skip-gmx-version-check`
There is no blanket `--no-doctor`. Each skip records into `provenance.json:doctor_skipped` so the audit trail is complete.

**R2-15** (wheel smoke test too shallow). **Accept.** Adding `mdagent self-test resources --json` that:
  - Resolves `schemas_dir()` to a real `Path`.
  - Loads every `*.schema.json` under it.
  - Computes the schemas-dir hash.
  - Resolves `mdagent._resources.skills` and lists every packaged skill.
  - Returns a JSON summary with `ok: bool` + counts.

The wheel test runs `mdagent self-test resources --json` against a clean venv-installed wheel and asserts `ok: true` + non-zero counts.

### 2.3 Skill bodies (final)

Each SKILL.md gets YAML frontmatter:

```yaml
---
name: md-run-workflow
description: ...
metadata:
  minimum_mdagent_version: "0.1.0"
  skill_version: "1.0.0"
---
```

The preflight in each skill body is exactly:

```bash
# Verify mdagent is installed and the right version.
mdagent doctor --json \
  --min-version 0.1.0 \
  --skill-name md-run-workflow \
  --skill-version 1.0.0 \
  > /tmp/mdagent_doctor.json || {
    cat /tmp/mdagent_doctor.json
    echo
    echo "Install or upgrade: uv tool install --force git+https://github.com/<user>/MDSimulationAgent@v0.1.0"
    echo "PATH: ensure '$(uv tool dir --bin 2>/dev/null || echo \"<uv tool bin dir>\")' is on PATH."
    exit 1
  }
```

Then the actual `mdagent run-workflow ...` call.

### 2.4 Doctor checks (final, config-aware)

The orchestrator calls `mdagent.doctor.check_for_run(cfg)` at the start of
`run_workflow()`. The check derives requirements from the config:

| Check | Triggered by config | Skip flag |
|---|---|---|
| gmx on PATH | always (any pipeline run needs gmx) | `--skip-gmx-version-check` skips the *version* part, but not presence |
| gmx version supported | always | `--skip-gmx-version-check` |
| prompt catalog matches gmx | always | (no skip — fail hard) |
| RCSB reachable | `input.pdb_id` set, no `input.structure_path` | `--skip-network-check` |
| Viewer renderable | `visualization.mode != disabled` AND `visualization.render in {png, both}` | `--skip-viewer-check` |

The CLI `mdagent doctor --json` runs the same engine with explicit
`--check-network` / `--check-viewers` / `--gmx-required` flags.

### 2.5 Wheel smoke test (final)

```python
# tests/test_wheel_install.py (slow + wheel marker)
def test_wheel_installs_and_self_tests(tmp_path):
    # 1) uv build
    subprocess.run(["uv", "build", "--out-dir", str(tmp_path / "dist")], check=True)
    wheel = next((tmp_path / "dist").glob("*.whl"))
    # 2) create venv
    venv = tmp_path / "venv"
    subprocess.run([sys.executable, "-m", "venv", str(venv)], check=True)
    pip = venv / "bin" / "pip"
    bindir = venv / "bin"
    subprocess.run([str(pip), "install", str(wheel)], check=True)
    # 3) introspect
    r = subprocess.run([str(bindir / "mdagent"), "--version"], capture_output=True, text=True, check=True)
    assert "mdagent " in r.stdout
    r = subprocess.run([str(bindir / "mdagent"), "self-test", "resources", "--json"], capture_output=True, text=True, check=True)
    data = json.loads(r.stdout)
    assert data["ok"] is True
    assert data["schemas"]["count"] >= 5
    assert data["skills"]["count"] >= 3
```
