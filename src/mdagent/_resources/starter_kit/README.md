# My MD-simulation project

Generated by `mdagent init-project`. This is a self-contained directory
that uses `mdagent` + Claude skills to run GROMACS MD simulations.

## One-time setup

If you don't already have `mdagent` and GROMACS:

```bash
# 1. Install uv (skip if you already have it)
brew install uv     # macOS — or: curl -LsSf https://astral.sh/uv/install.sh | sh

# 2. Install mdagent (pinned to a git tag for reproducibility)
uv tool install --force git+https://github.com/mjayadharan/MDSimulationAgent@v0.1.0

# 3. Install GROMACS (only needed for steps past prep)
brew install gromacs    # macOS

# 4. (Optional) install the Claude skills globally as well
mdagent install-skills --user
```

The skills are already installed *for this project* under
`.claude/skills/` — Claude Code finds them automatically when you open
a session here. `--user` is only needed if you want them everywhere.

## First run

```bash
./verify.sh                 # structural + schema checks (offline, no gmx required)
./verify.sh --run-smoke     # actually runs the pipeline (~2 min)
```

A successful `--run-smoke` writes `runs/smoke/REPORT.md` with
`readiness: **ready**`.

## Run a simulation directly

```bash
# Offline (uses the bundled 1AKI structure):
mdagent run-workflow --runs-root ./runs --config ./run_configs/lysozyme_short.json --run-id demo

# Inspect:
mdagent inspect --run-root ./runs/demo
```

## Run via Claude Code (natural language)

Open a Claude Code session in this directory and ask, for example:

> *"Set up lysozyme in water and minimize it."*
> *"Prep 1AKI for simulation, then equilibrate at 300 K and 1 bar."*
> *"Run the lysozyme tutorial config and tell me when EM converges."*

Claude routes to the `md-run-workflow` skill (or `md-prep-structure` /
`md-visualize` as appropriate).

## Editing configs

Configs are plain JSON validated against the schema at
`src/mdagent/_resources/schemas/v0.1.0/run_config.schema.json` in the
installed `mdagent` package. Common knobs:

- `force_field`: `oplsaa` (default), `amber99sb-ildn`, `charmm36-jul2022`.
- `water_model`: `spc`, `spce`, `tip3p`, `tip4p` (must match FF allowlist).
- `box.geometry`: `dodecahedron` (default), `cubic`, `octahedron`.
- `box.padding_nm`: nanometers around the solute. Default 1.0.
- `ion_strategy.mode`: `neutralize_only` (default) or `physiological_salt` (set `salt_M: 0.15`).
- `em.step_cap`: bump if EM didn't converge.
- `nvt.nsteps` / `npt.nsteps`: equilibration length. 50000 × 2 fs = 100 ps.
- `production.nsteps`: production length. 500000 × 2 fs = 1 ns.
- `production.enabled`: `false` to stop after NPT (prep-only).
- `pipeline_mode`: `tutorial_reproduction` (default) or `general_md_prep`
  (drives `pdb2gmx -inter` for per-residue protonation).

## Refreshing skills after upgrading mdagent

```bash
uv tool install --force git+https://github.com/mjayadharan/MDSimulationAgent@<newer_tag>
mdagent install-skills --project . --force
```

## What this directory does NOT do

- Cloud / HPC execution (RemoteExecutor not yet wired).
- Production-length runs on a laptop in seconds — adjust `nsteps`
  accordingly for your hardware.
- PROPKA-driven protonation in general mode (currently fixed pH-7 defaults).
